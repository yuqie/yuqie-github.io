使用scp远程传输文件

Posted on 2019-09-17 In Text

scp是 secure copy的缩写, scp是linux系统下基于ssh登陆进行安全的远程文件拷贝命令。linux的scp命令可以在linux服务器之间复制文件和目录.

scp命令基本格式：

scp [-1246BCpqrv] [-c cipher] [-F ssh_config] [-i identity_file]
[-l limit] [-o ssh_option] [-P port] [-S program]
[[user@]host1:]file1 [...] [[user@]host2:]file2

简略版：scp [[user@]host1:]file1 [...] [[user@]host2:]file2

              scp user@host1:传输文件或目录 user@host2:目标目录

注：复制目录时要加-r

       可以传输多个文件或目录

scp命令的参数说明:

-1
强制scp命令使用协议ssh1

-2
强制scp命令使用协议ssh2

-4
强制scp命令只使用IPv4寻址

-6
强制scp命令只使用IPv6寻址

-B
使用批处理模式（传输过程中不询问传输口令或短语）

-C
允许压缩。（将-C标志传递给ssh，从而打开压缩功能）

-p 保留原文件的修改时间，访问时间和访问权限。

-q
不显示传输进度条。

-r
递归复制整个目录。

-v 详细方式显示输出。scp和ssh(1)会显示出整个过程的调试信息。这些信息用于调试连接，验证和配置问题。

-c cipher
以cipher将数据传输进行加密，这个选项将直接传递给ssh。

-F ssh_config
指定一个替代的ssh配置文件，此参数直接传递给ssh。

-i identity_file
从指定文件中读取传输时使用的密钥文件，此参数直接传递给ssh。

-l limit
限定用户所能使用的带宽，以Kbit/s为单位。

-o ssh_option
如果习惯于使用ssh_config(5)中的参数传递方式，

-P port 注意是大写的P, port是指定数据传输用到的端口号

-S program
指定加密传输时所使用的程序。此程序必须能够理解ssh(1)的选项


awk操作

Posted on 2019-09-15 In Text

1. 添加行号：awk '{print NR "\t" $0}'  input.txt >output.txt

2. 提取指定列：awk 'print{$1"\t"$2"\t"$5}' input.txt > output.txt #若干列 #\t 表示制表符

3. 提取指定行指定列：cat input.txt|awk 'NR==2{print $3}' > output.txt # 第二行第三列

4. 指定列之间运算：awk '{ for(i = 1;i<=NF;i++){printf("%-15s",$i);}; if(NR!=1) print ($3 - $4); else print("D-Value"); } ' test.txt  #NF：number of file；NR：number of row

转载：https://www.cnblogs.com/djoker/p/9408716.html

awk:
　　awk是一个强大的文本分析工具，相对于grep的查找，sed的编辑，
　　awk在其对数据分析并生成报告时，显得尤为强大。
　　简单来说awk就是把文件逐行的读入，以空格为默认分隔符将每行切片，切开的部分再进行各种分析处理。
　　awk有3个不同版本: awk、nawk和gawk，未作特别说明，一般指gawk，gawk 是 AWK 的 GNU 版本。
　　awk其名称得自于它的创始人 Alfred Aho 、Peter Weinberger 和 Brian Kernighan 姓氏的首个字母。
　　实际上 AWK 的确拥有自己的语言： AWK 程序设计语言 ， 三位创建者已将它正式定义为“样式扫描和处理语言”。
　　它允许您创建简短的程序，这些程序读取输入文件、为数据排序、处理数据、对输入执行计算以及生成报表，还有无数其他的功能。

awk工作流程:
　　读入有"\n"换行符分割的一条记录,然后把记录按照指定的域分隔符划分域,填充域
　　$0表示所有的域,$1表示第一个域,以此类推
　　默认域分隔符是空白键或者tab键
　　

-F:指定域分隔符

cat /etc/passwd | awk -F":"'{print $1}'

中间可以用\t来填充

cat /etc/passwd | awk -F":"'{print $1"\t"$7}'

当有BEGIN,END时AWK工作流程:
　　先执行begin,然后读取文件,读入有\n换行符分割的第一条记录,然后安装
　　指定的域分隔符划分域,填充域,$0表示所有的域,$1表示第一个域,以此类推
　　随后开始执行模式所对应的动作action
　　最后都执行完,最后执行end操作

　cat /etc/passwd | awk -F':''BEGIN {print "username bash"} {print $1","$7} END {print "run over"}'

      输出内容需要用""包含,其中BEGIN,END都是关键字,必须大写

awk内置变量:
　　awk中同样定义了很多内置变量，我们可以直接像使用普通变量一样使用他们，由于awk的版本众多，有些内置变量并不是得到所有awk版本的支持。

　　说明：[A][N][P][G]表示支持该变量的工具，[A]=awk、[N]=nawk、[P]=POSIXawk、[G]=gawk

  

$n 当前记录的第n个字段，比如n为1表示第一个字段，n为2表示第二个字段。 

　　$0 这个变量包含执行过程中当前行的文本内容。

　　[N] ARGC 命令行参数的数目。

　　[G] ARGIND 命令行中当前文件的位置（从0开始算）。

　　[N] ARGV 包含命令行参数的数组。

　　[G] CONVFMT 数字转换格式（默认值为%.6g）。

　　[P] ENVIRON 环境变量关联数组。

　　[N] ERRNO 最后一个系统错误的描述。

　　[G] FIELDWIDTHS 字段宽度列表（用空格键分隔）。

　　[A] FILENAME 当前输入文件的名。

　　[P] FNR 同NR，但相对于当前文件。

　　[A] FS 字段分隔符（默认是任何空格）。

　　[G] IGNORECASE 如果为真，则进行忽略大小写的匹配。

　　[A] NF 表示字段数，在执行过程中对应于当前的字段数。

　　[A] NR 表示记录数，在执行过程中对应于当前的行号。

　　[A] OFMT 数字的输出格式（默认值是%.6g）。

　　[A] OFS 输出字段分隔符（默认值是一个空格）。

　　[A] ORS 输出记录分隔符（默认值是一个换行符）。

　　[A] RS 记录分隔符（默认是一个换行符）。

　　[N] RSTART 由match函数所匹配的字符串的第一个位置。

　　[N] RLENGTH 由match函数所匹配的字符串的长度。

　　[N] SUBSEP 数组下标分隔符（默认值是34）。

step3. 配置xshell,连接-》SSH -> 隧道-》在x11 转移中设置xdisplay为localhost:0.0

step4, 配置完成后，重新连接服务器，

mdzs很火的时候，在晋江追过小说，当时看到义城后因为三次元太忙，没有追下去，在之后就没怎么关注过魔道了，也不知道魔道要电视剧化。

陈情令是六月底开播的，当时有很多负面评价，很多营销号带节奏，印象最深的一条是：陈情令的片头和坠崖的场景很多借鉴了琅琊榜，琅琊榜可是最近七八年来我最喜欢的大陆古装剧，导致我对这部剧的最初印象不太好。至于孟情令，蓝忘机扮相太丑之类的，是在开始追这部剧后才有所了解的。

未知全貌，不予置评，一直是我坚持的原则，当时那么多负面评价，让我觉得仿佛这部剧挡了谁家的道。刚好六月底我手头的事情暂时告一段落，就想着要不看看，是不是真得那么差（我是一个当年还追过一两集云中歌的人，当时就是为了看看AB的演技能有多差，捂脸）。就这样，在某个晚上，一口气二倍速看了十集的我入了陈情令的坑。仔细想想，可能是肖战帅气的脸庞和一点点原著小说的印象让我撑过前十集。

就这样我过上追剧的日子，每周最开心的就是周一啦，因为可以连着三天看剧，也看着豆瓣评分一点点涨上去。

作为一部古装仙侠偶像剧，服化道过关（虽然很多场景看上去很美，却有一种伪古风的赶脚，当然作为一部没有时代背景架空剧，已经很好啦），众主创演技及格（博机越看越顺眼，基本上有姓名的角色演技都还凑合，比如欧阳子真，哈哈哈，昨天发现这个小伙子挺帅的；去年的镇魂，除了两个主演，其他角色真是一言难尽，后来看完也是为了有始有终，都开三倍速了），特效马马虎虎也不错，除了开头众人强阴虎符那一块（最近最喜欢的特效是香蜜里面大龙的尾巴，天帝玉和旭凤大战场景）。

但是，群演实在是太少了，说好的不夜天三千修士呢，还有攻打温家的时候，基本上仙门所有力量全部出动了吧，不说几千人了，怎么也得凑个大几百人吧，看的时候感觉一个家族就二十来人，除了有名有姓的家族，其他家族呢？本来很大的场面，看上去像过家家一样。另外，阿令给我的感觉就是没有武术指导，打戏太水了吧，是在逗我玩么，除了几个主要角色的特定场景看上去还不错，其他的时候真是一言难尽。最后，感觉阿令的剪辑有点跳，或者说节奏把握得不太好。比如火烧云深不知处时，蓝忘机断了腿，为什么没有放出来相应片段呢，蓝湛的腿直接就断了，要不是追剧期间把小说刷了一遍，谁知道前因后果呢。

昨晚看从乱葬岗到莲花坞和观音庙的这两场戏时，节奏衔接就很好，剧情也很有张力。尤其是观音庙这场，哭瞎了，处处都是刀。关于最终的结局，我个人觉得可以接受，有人会说蓝二接任仙督，不符合他的人设。可是汪叽不是恋爱脑呀，蓝二有他的责任与担当，在仙门刚刚经历一次动荡之后，留下来，难道不是让蓝二的形象更立体么。恰当的留白给人遐想的空间，最后wifi和汪叽可能一起隐居，一起夜猎，也可能是其他的发展。（所以有大大写剧版忘羡么，笑哭）。

陈情令里面的主要角色都演得很好，虽然表情包满天飞（汪叽，羡羡，师姐，舅舅，阿凌，金子轩，温晁温主任，娇娇，蓝启仁，思追和景仪，谁都没有逃过，哈哈哈）。毋庸置疑，阿令可以拍得更好，所以对于打戏，剧情节奏等方面的不足，我才会觉得惋惜，因为他们有一众够敬业的演员，他们值得更好。

合卷之后，闭上眼睛，脑海里浮现出一幅幅书中描写的画面，剧中的场景，以及最后简单但是别有意味的结尾。感谢阿令。
